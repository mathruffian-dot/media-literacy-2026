# SRT 清字規則

## 絕對不可做（違反即失敗）

1. **不可動時間碼**：`00:01:23,456 --> 00:01:27,890` 這行逐字元不變
2. **不可合併段落**：raw 有 N 段，clean 必須也是 N 段
3. **不可拆分段落**：一段就是一段，不許變兩段
4. **不可移動文字**：某句話在第 5 段就留在第 5 段，不許挪到第 6 段
5. **不可增刪語意**：不加入 raw 沒有的內容，不刪除 raw 的內容（除了口頭禪）

## 應該做

### 錯字修正
Whisper 中文常見誤判：
- 同音字：「在」/「再」、「的」/「得」/「地」、「做」/「作」
- 專有名詞誤判：見 `vocabulary.md`
- 音近字：「然後」被聽成「染後」之類

### 標點補充
- 段內加入適當逗號、句號、問號
- 中文標點用全形（，。？！）
- 括號、引號視語意補上

### 口頭禪移除（謹慎）
可移除**不影響資訊的**贅詞：
- 「那個那個」→ 刪（若重複無意義）
- 「對對對」→ 保留一個「對」
- 「就是說」→ 視情況保留或刪
- 「然後然後」→ 保留一個「然後」

**但若移除後段落變空，就保留原樣**（寧可保留贅詞也不要空段）。

### 斷句順暢
段內可調整標點位置讓語感順暢，但：
- 不改變字序
- 不新增轉折詞
- 保留原始語氣

## 工作流（逐段處理）

對每一段 SRT：
```
原始段落：
  42
  00:05:12,340 --> 00:05:15,670
  那個我們今天來講這個 claw code 桌面版

處理後：
  42
  00:05:12,340 --> 00:05:15,670
  我們今天來講 Claude Code 桌面版。
```

注意：
- 段號 `42` 不變
- 時間碼行不變
- 只改第 3 行（文字）

## 自我檢查

清完一段，問自己：
- [ ] 這段的時間碼跟 raw 完全一樣嗎？
- [ ] 這段的資訊量跟 raw 相同嗎？
- [ ] 我有沒有不小心把下一段的內容挪進來？

若清到一半發現某段資訊應該屬於上/下段（Whisper 斷錯地方），**不要動**，保留原樣交付。時間碼正確比斷句完美重要。
